# Component UI Spec

Project: CallBotCompare

## 1. Header
### Purpose
Primary site navigation and persistent CTA.

### Style
- slim height
- cream or translucent light background
- subtle border bottom
- sticky on scroll

### Includes
- wordmark/logo
- nav links
- CTA button

## 2. Hero block
### Purpose
Explain the site instantly and drive the first action.

### Style
- two-column desktop layout
- left aligned copy
- right side comparison preview card
- warm and structured

### Notes
- no decorative illustration dependency
- comparison card should make the product concept obvious

## 3. Methodology strip
### Purpose
Show credibility and evaluation logic.

### Style
- compact grid
- muted label chips
- minimal icons if any

## 4. Vendor card
### Purpose
Preview a vendor in a consistent and scannable way.

### Required fields
- logo/name
- summary
- top badges
- score
- CTA

### Style
- white card
- medium radius
- low shadow
- strong spacing

## 5. Use-case card
### Purpose
Send visitors into vertical landing pages.

### Includes
- vertical title
- one-sentence context
- arrow or text link

### Style
- slightly tinted background variation allowed
- cards should not all feel identical

## 6. Feature tile
### Purpose
Link to compare filters and feature-led pages.

### Style
- simple tile or mini card
- icon optional
- short text only

## 7. Comparison table
### Purpose
Core product utility module.

### Required columns
- vendor
- best for
- pricing
- booking
- CRM integration
- human handoff
- after-hours
- score
- CTA

### Style
- highly readable
- sticky top header on desktop
- smooth horizontal scroll fallback on smaller widths
- alternating rows acceptable

## 8. Vendor detail summary card
### Purpose
Top-level vendor snapshot.

### Includes
- vendor name/logo
- summary
- score
- best for
- pricing snapshot
- CTA button

## 9. Score breakdown block
### Purpose
Show how a vendor performs across key dimensions.

### Style
- list or bars
- understated, not gamified
- useful labels only

## 10. Pros and cons block
### Purpose
Fast evaluation aid.

### Style
- two-column desktop layout
- simple icons/checks
- avoid emoji feel

## 11. CTA banner
### Purpose
Move users into quiz/contact/referral action.

### Style
- pale sage or cream tint
- concise copy
- one primary action, one optional secondary

## 12. FAQ accordion
### Purpose
Support clarity and SEO.

### Style
- clean borders
- generous spacing
- simple chevron behavior

## 13. Quiz stepper
### Purpose
Guide users through recommendation flow.

### Style
- clear progress indicator
- one primary question per step
- low-friction controls
- trust-building microcopy

## 14. Lead form
### Purpose
Capture concierge recommendation requests.

### Style
- calm, professional
- not overlong visually
- grouped fields
- optional help text

## 15. Footer
### Purpose
Close the experience cleanly and support trust.

### Includes
- site description
- link groups
- disclosure/privacy
- contact path

## Shared design rules
- keep spacing consistent
- keep text readable
- use badges sparingly
- avoid oversized iconography
- prefer structure over decoration
- every component should look intentionally designed, not copied from a UI kit demo
