# Homepage Wireframe

Project: CallBotCompare

## Goal
Design a homepage that feels trustworthy, useful, and intentionally structured.

The page should behave like a buyer guide with product-style clarity.

## Desktop wireframe

### 1. Header
Layout:
- left: wordmark/logo
- center/right: nav links
- far right: primary CTA button

Nav links:
- Compare
- Vendors
- Guides
- Quiz

CTA:
- Get recommendations

Notes:
- sticky on scroll
- slim, clean, not oversized
- subtle bottom border on scroll

---

### 2. Hero
Layout:
- two-column layout
- left side: message and actions
- right side: comparison preview card

#### Left column
- eyebrow: Buyer guide for AI receptionist software
- H1: Compare AI receptionist software without wasting weeks on demos.
- supporting text
- primary CTA: Compare tools
- secondary CTA: Take the quiz
- short proof bullets

#### Right column
Mock module:
- mini comparison panel with 3 vendors
- columns for score, booking, integrations, best for
- subtle card stack or framed table

Notes:
- no decorative fluff
- this area should instantly explain what the site does

---

### 3. Trust / methodology strip
Layout:
- full-width horizontal section
- 4 to 5 compact blocks

Blocks:
- call handling
- integrations
- booking workflows
- handoff and escalation
- setup complexity

Notes:
- can include tiny labels or score-style chips
- should feel methodological, not salesy

---

### 4. Top picks section
Layout:
- heading + short intro
- 3 featured vendor cards in a row

Each card:
- logo/name
- one-line summary
- 2 or 3 muted badges
- score
- CTA

Notes:
- this is not the full directory
- this is a curated entry point

---

### 5. Compare by use case
Layout:
- 2 rows of 3 cards or responsive grid

Cards:
- Law firms
- Home services
- Medical offices
- Contractors
- Real estate teams
- Small business

Card contents:
- title
- one sentence
- arrow link

Notes:
- cards should use soft background rhythm, not identical bland boxes

---

### 6. Compare by feature
Layout:
- left: short explanation
- right: feature grid or compact tiles

Features:
- appointment booking
- CRM integration
- human handoff
- multilingual support
- after-hours coverage
- lead qualification
- analytics
- healthcare-friendly workflows

Notes:
- useful for quick scan
- should connect to filterable compare experience

---

### 7. How we evaluate
Layout:
- editorial section
- heading and body on left
- scoring framework visual on right

Scoring visual:
- weighted list or clean chart
- not a flashy infographic

Notes:
- reinforces trust
- should feel transparent and calm

---

### 8. Concierge recommendation CTA
Layout:
- centered callout block with warmer background tint

Contents:
- heading
- short body copy
- CTA: Get a recommendation
- optional secondary: Talk to us

Notes:
- should feel advisory, not desperate

---

### 9. Latest comparisons and guides
Layout:
- tabbed or two-column block
- 3 comparison cards
- 3 guide cards

Notes:
- keeps homepage fresh
- supports internal linking and SEO depth

---

### 10. FAQ
Layout:
- clean accordion
- 5 to 7 questions

Notes:
- useful questions only
- no filler FAQ spam

---

### 11. Footer CTA
Layout:
- short closing section
- primary CTA + secondary CTA

Copy direction:
- Compare tools with a clearer process

---

### 12. Footer
Sections:
- logo and short site description
- compare links
- guides links
- company/contact links
- disclosure/privacy links

## Mobile wireframe

### Mobile order
1. Header
2. Hero
3. Methodology strip
4. Top picks
5. Compare by use case
6. Compare by feature
7. How we evaluate
8. Concierge CTA
9. Latest content
10. FAQ
11. Footer CTA
12. Footer

## Visual rhythm rules
- Alternate white and cream sections
- Break up card grids with editorial sections
- Use tables sparingly but prominently
- Keep every section visually distinct enough to avoid template sameness

## Key instruction for Claude Code
The homepage should feel custom and deliberate. It should not resemble a generic AI SaaS hero followed by six identical Tailwind card grids.
